### TCP 服务

1. Application 应用层: HTTP、SMTP、IMAP
2. Presentation 表示层: 加密/解密等
3. Session 会话层: 通信连接/维持会话
4. Transport 传输层: TCP/UDP
5. Network 网络层: IP
6. Data 链路层: 网络特有的链路接口
7. Physical 物理层: 网络物理硬件

TCP 是面向连接的协议，其显著的特征是在传输之前需要 3 次握手形成会话。

只有会话形成之后，服务器端和客户端之间才能发送数据。**在创建会话的过程中，服务器端和客户端分别提供一个 socket，这两个 socket 共同形成一个连接**。服务器和客户端则通过套接字实现两者之间的操作。

#### TCP 服务的事件

服务器可以同时与多个客户端保持连接，队友每个连接而言是典型的可读可写 stream 对象。

#### 构建 WebSocket 服务

WebSocket 实现了客户端与服务端之间的长连接，而 Node 事件驱动的方式十分擅长与大量的客户端保持并发连接。
